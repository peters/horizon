Make the minimap answer “where should I look?”

Temporary exploration This pre-implementation artifact is retained only for draft-PR context. Motion was evaluated and rejected: the accepted workspace spotlight and attention cues are fully static, with no pulse or continuous repaint.

The current minimap is good at preserving board geometry, but workspace identity colors, focus, labels, panels, and the viewport all compete at the same tiny scale. These concepts separate the two important questions: which workspace is active, and which workspace or panel needs attention. Each preview uses the same simulated board and the same Horizon dark theme tokens.

Concept 01 Recommended

Workspace spotlight

A neutral blue double ring marks the active workspace. Exact panels get shaped attention beacons, while a count pill keeps the signal legible when the panel becomes too small.

2× design preview · target 320 × 180
ActiveDouble outline, brighter fill, and an ACTIVE tab.
AttentionPanel beacon plus workspace count. High = !, done = ✓, info = i.
Best forA familiar default that adds strong state without changing minimap layout.
Concept 02 Dense scale

Orthogonal edge rails

Active and attention use different edges: a left-side focus rail and a top-side severity rail. Their width stays constant even when a workspace collapses into a narrow sliver.

2× design preview · constant-pixel cues
ActiveBlue left rail and corner brackets; independent of workspace accent.
AttentionRed, green, or cyan top rail with icon and count at its endpoint.
Best forVery dense boards where fills, glows, and text disappear at minimap scale.
Concept 03 Attention-first

Attention radar

The spatial map stays quiet while numbered urgent beacons connect to a compact workspace rail. It trades some canvas area for fast scanning and larger, more reliable click targets.

2× design preview · map + status rail
ActivePersistent target glyph and underline on the workspace chip.
AttentionNumbered urgent pins map to rail chips; completed work uses a check.
Best forAgent-heavy boards where triage order matters more than maximum map area.

Decision guide

All three can be built from current board state. The main decision is how much of the minimap should remain a pure spatial overview versus becoming an operational status surface.

Use more than color Ring or rail for active; !, ✓, and i plus a count for severity. This remains readable with color-vision differences.
Collapse adaptively Paint panel-level markers only above a size threshold. Otherwise show one workspace count outside the geometry.
Stay static Attention cues do not pulse or animate. A normal UI event refreshes time-windowed state without adding a repaint loop.
